1. Create `tests/image-paste.spec.ts` covering paste and upload flow:

   **Test helpers:**
   - Create a small test PNG buffer in the test setup. Use a 1x1 pixel red PNG as a minimal valid image (Buffer of known bytes). This avoids needing a test fixture file.
   - Helper function `pasteImage(page, buffer, mimeType)` that dispatches a ClipboardEvent with DataTransfer containing the image:
     ```typescript
     // Playwright can't directly simulate system clipboard paste.
     // Instead, use the API endpoint directly for most tests, and
     // for the paste handler test, use page.evaluate to dispatch
     // a synthetic ClipboardEvent.
     async function pasteImageOnPage(page: Page, imageBase64: string, mimeType: string) {
       await page.evaluate(({ base64, mime }) => {
         const binary = atob(base64);
         const bytes = new Uint8Array(binary.length);
         for (let i = 0; i < binary.length; i++) bytes[i] = binary.charCodeAt(i);
         const blob = new Blob([bytes], { type: mime });
         const file = new File([blob], 'test.png', { type: mime });

         const dt = new DataTransfer();
         dt.items.add(file);

         const event = new ClipboardEvent('paste', {
           clipboardData: dt,
           bubbles: true,
           cancelable: true,
         });
         document.dispatchEvent(event);
       }, { base64: imageBase64, mime: mimeType });
     }
     ```
   - Helper `createTestPng()` that returns a base64-encoded minimal PNG.
   - Use unique slug per test: `test-img-${Date.now()}-${Math.random().toString(36).slice(2, 8)}`.

   **Tests (follow naming convention with requirement ID prefix):**

   - `IMG-01: pasting an image creates a visible thumbnail in the grid` -- Navigate to pad, paste a test PNG via the synthetic clipboard event, wait for `.image-card[data-status="loaded"]` to appear. Assert image element exists within the card.

   - `IMG-01: pasting text still works in the textarea` -- Navigate to pad, focus textarea, type text, verify textarea value contains the text. Then simulate a text paste (not image) and verify textarea receives the text. This ensures the paste handler does not intercept text pastes.

   - `IMG-01: only the first image is processed on multi-image paste` -- Paste an event with two image items in the DataTransfer. Assert only one skeleton/image card appears (not two).

   - `IMG-02: uploaded image is accessible via API as WebP` -- Navigate to pad, upload via API (POST /api/pads/{slug}/images with FormData), then GET the returned image URL. Assert response Content-Type is 'image/webp' and body is non-empty.

   - `IMG-03: images have loading="lazy" attribute` -- Upload an image, wait for it to appear, assert `img[loading="lazy"]` is present in the DOM.

   - `IMG-04: uploading an oversized image shows error card` -- Create a buffer >5MB (fill with random data or zeros). POST to the upload API. Assert 413 status. On the frontend: paste a file that claims to be >5MB (client-side check). Assert `.image-card-error` appears with text containing "too large".

   - `IMG-04: pad quota rejection shows error message` -- This is harder to test e2e without uploading 100MB. Use the API directly: upload many small images until quota is exceeded, assert 413 response with "quota exceeded" message. Frontend test: verify the error card text.

   - `IMG-01: pressing Escape closes the lightbox` -- Open lightbox, press Escape, assert `.lightbox-overlay` is not visible.

   - `IMG-01: delete button removes image from grid` -- Upload an image via API, navigate to pad, hover over the image card, click `.delete-btn`, assert the image card is removed from the DOM. Verify the API confirms deletion (GET the image URL returns 404).
