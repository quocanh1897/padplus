Build the core pad editor experience: server-side rendered text, auto-save, optimistic concurrency, and conflict resolution UX.

**Server load function (`src/routes/[...slug]/+page.server.ts`):**
- Import `getPadBySlug`, `createPad` from `$lib/server/pads`
- `load({ params })`: extract `params.slug` (the full catch-all path)
- If pad exists, return `{ slug, content, version }`. If not, create it and return the new pad.
- This ensures the textarea always has content in SSR HTML (even if empty string for new pads).
- IMPORTANT: Validate slug -- reject slugs starting with `_` or `api`, containing file extensions (`.ico`, `.js`, `.css`). Use SvelteKit's `error(400)` for invalid slugs.

**API endpoint (`src/routes/api/pads/[...slug]/+server.ts`):**
- Export `PUT` handler: parse JSON body `{ content: string, version: number }`
- Validate inputs: content must be string, version must be number
- Call `savePad(slug, content, version)`
- If no conflict: return `200` with `{ version: newVersion, updatedAt }`
- If conflict: return `409` with `{ error: 'conflict', message: '...', content: serverContent, version: serverVersion }`
- Use SvelteKit's `json()` and `error()` helpers

**Pad editor page (`src/routes/[...slug]/+page.svelte`):**
- Use Svelte 5 runes: `let { data } = $props()`
- Create reactive state: `let content = $state(data.content)`, `let version = $state(data.version)`, `let saveStatus = $state<'saved' | 'saving' | 'unsaved' | 'conflict' | 'error'>('saved')`, `let isSaving = $state(false)`, `let conflictData = $state<{ serverContent: string; serverVersion: number } | null>(null)`
- CRITICAL for SSR: Render `<textarea>{data.content}</textarea>` with the server data directly in the template so it appears in HTML before JS loads. After hydration, bind to the reactive `content` state. Use `value={content}` and `oninput` handler to track changes.
- Auto-save logic (inside `$effect` or guarded with `browser` check):
  - Import `debounce` from `$lib/utils/debounce`
  - On content change (not on initial load), set `saveStatus = 'unsaved'`
  - Debounced save function (400ms): sets `saveStatus = 'saving'`, calls `fetch PUT /api/pads/${slug}` with `{ content, version }`
  - On success: update `version` from response, set `saveStatus = 'saved'`
  - On 409: set `saveStatus = 'conflict'`, store `conflictData` with server content/version, PAUSE auto-save
  - On other error: set `saveStatus = 'error'`
  - Use `isSaving` flag to prevent overlapping requests (Pitfall 5 from research)
- Conflict resolution:
  - When `conflictData` is not null, show `<ConflictBanner>` at top of editor
  - "Overwrite" action: call save with `conflictData.serverVersion` as expected version, clear conflict state, resume auto-save
  - "Copy my version & reload": copy content to clipboard via `navigator.clipboard.writeText()`, then `invalidateAll()` or `location.reload()` to get server state
- Textarea styling: `width: 100%`, `height: calc(100vh - headerHeight)`, `resize: none`, `border: none`, `outline: none`, `padding: var(--space-lg)`, `font-family: var(--font-body)`, `font-size: var(--font-size-lg)`, `background: var(--color-bg)`, `color: var(--color-text)`. Sans-serif font per user decision.
- Layout: full width, editor dominates the page, no visible chrome around the editor per user decision.
- Responsive: on mobile, reduce padding, ensure textarea is still full-width and usable (CORE-06).
