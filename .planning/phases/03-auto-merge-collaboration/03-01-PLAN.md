**3. Create src/lib/server/merge.ts:**
Create a merge module wrapping node-diff3's `diff3Merge` function (NOT the `merge` function -- `diff3Merge` returns structured ok/conflict blocks instead of embedding conflict markers in text).

Export a `MergeResult` interface:
```typescript
export interface MergeResult {
  success: boolean;  // true = clean merge, false = had conflicts (best-effort concatenation)
  content: string;   // merged text content
}
```

Export a `mergeText(base, server, client)` function:
- Split all three inputs on `'\n'` into line arrays
- Call `diff3Merge(clientLines, baseLines, serverLines, { excludeFalseConflicts: true })`
- Iterate over regions: push `region.ok` lines for clean regions, for `region.conflict` push both `region.conflict.a` (client) then `region.conflict.b` (server) lines as best-effort concatenation per CONTEXT.md ("let the merge algorithm decide -- result may concatenate both versions at the conflict point")
- Track `hasConflict` boolean
- Join merged lines with `'\n'` and return `{ success: !hasConflict, content: joined }`

**4. Extend src/lib/server/pads.ts:**
Keep the existing `savePad` function signature but change its internal behavior to support merge mode:

Add new prepared statements:
- `getModeStmt`: `SELECT collaboration_mode, content, base_content, version FROM pads WHERE slug = ?`
- `updateWithBaseStmt`: Same as existing `updateStmt` but also sets `base_content = ?` (the new content, since after a successful direct save the saved content IS the new base)
- `forceUpdateStmt`: `UPDATE pads SET content = ?, base_content = ?, version = ?, updated_at = datetime('now') WHERE slug = ?` (for writing merge results)

Modify `savePad()`:
- First, try the normal version-checked update (using `updateWithBaseStmt` which also sets base_content)
- If `result.changes === 0` (version mismatch):
  - Read the current pad with `getModeStmt`
  - If `collaboration_mode !== 'auto-merge'`: return `{ type: 'conflict', pad }` (same as before)
  - If `collaboration_mode === 'auto-merge'`: call `mergeText(current.base_content, current.content, content)`, then save the merged result using `forceUpdateStmt` inside a `db.transaction()` (wrapping the read + merge + write to prevent race conditions per Research Pitfall 3). Return `{ type: 'merged', pad, hadConflicts: !mergeResult.success }`
- If `result.changes > 0`: return `{ type: 'saved', pad }` (clean save)

Change the return type from `{ conflict: boolean; pad: Pad }` to:
```typescript
export type SaveResult =
  | { type: 'saved'; pad: Pad }
  | { type: 'merged'; pad: Pad; hadConflicts: boolean }
  | { type: 'conflict'; pad: Pad };
```

IMPORTANT: The entire merge path (read current pad + compute merge + write result) MUST be wrapped in `db.transaction()` to prevent race conditions. better-sqlite3's transaction uses `BEGIN IMMEDIATE` by default which acquires a write lock before reading.

Add `updateCollaborationMode(slug, mode)` function:
- Prepared statement: `UPDATE pads SET collaboration_mode = ? WHERE slug = ?`
- Validate mode is `'last-save-wins'` or `'auto-merge'`
- Returns the updated pad (re-fetch after update)

Also update the `insertStmt` to set `base_content = ''` (it already defaults to '' from the migration, but be explicit if needed).

- `type: 'saved'`: Return 200 with `{ version, updatedAt }` (unchanged from before)

The `content` field contains the merged text. `hadConflicts` tells the client whether overlapping edits were concatenated (best-effort).

**2. Create src/routes/api/pads/[...slug]/mode/+server.ts:**

Create a PATCH endpoint for changing the collaboration mode of a pad.

```typescript
import { json, error } from '@sveltejs/kit';
import { updateCollaborationMode } from '$lib/server/pads';
import type { RequestHandler } from './$types';

export const PATCH: RequestHandler = async ({ params, request }) => {
  const slug = params.slug;
  if (!slug) error(400, 'Slug is required');

  const body = await request.json();
  const { mode } = body as { mode: unknown };

  if (mode !== 'last-save-wins' && mode !== 'auto-merge') {
    error(400, 'Mode must be "last-save-wins" or "auto-merge"');
  }

  const pad = updateCollaborationMode(slug, mode);
  return json({ mode: pad.collaboration_mode });
};
```

Validate that mode is one of the two valid strings. Return the updated mode in the response. If the pad doesn't exist, `updateCollaborationMode` should return an appropriate error (the function should throw or return null if slug not found).
