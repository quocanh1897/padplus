<task type="auto">
  <name>Task 1: Docker E2E test script and Playwright test suite</name>
  <files>
    test-docker.sh,

  </files>
  <action>
1. Create `test-docker.sh` -- a shell script that:
   - Sets `-e` (exit on error) and traps cleanup on EXIT
   - Defines container name `padplus-e2e-test` and port `3198` (avoid conflicts with dev server on 5173 or any running instance on 3000)
   - Builds the Docker image: `docker build -t padplus-e2e .`
   - Starts the container: `docker run -d --name $CONTAINER -p $PORT:3000 padplus-e2e`
   - Waits for health check (poll `curl -sf http://localhost:$PORT/health` with up to 30s timeout, 1s intervals)
   - Runs Playwright tests: `DOCKER_TEST_URL=http://localhost:$PORT npx playwright test tests/docker.spec.ts --project=chromium`
   - Cleanup function (in trap): `docker stop $CONTAINER 2>/dev/null; docker rm $CONTAINER 2>/dev/null`
   - Make executable: `chmod +x test-docker.sh`

2. Create `tests/docker.spec.ts` with Playwright tests. The test file should:
   - Read `DOCKER_TEST_URL` from `process.env` with fallback to `http://localhost:3198`
   - Use `test.describe('Docker Deployment')` grouping
   - Use unique timestamped slugs per test to avoid cross-test interference (pattern: `docker-test-${Date.now()}`)

   **Test cases:**

   a. **"app is accessible after docker run"**
      - Navigate to the base URL (landing page)
      - Expect the page to contain the PadPlus landing page elements (URL input bar)
      - Verify HTTP 200 response

   b. **"health check endpoint responds"**
      - Fetch `${baseURL}/health` using `request.get()`
      - Expect 200 status
      - Expect response body to contain `{ status: 'ok' }`

   c. **"can create and save a pad"**
      - Navigate to `/${slug}`
      - Fill textarea with test content
      - Wait for auto-save (600ms debounce + buffer)
      - Reload the page
      - Verify textarea contains the saved content

   d. **"data persists across container restart"**
      - This test requires container restart. Use `execSync` to run:
        `docker restart padplus-e2e-test`
      - Wait for health check to return (poll with retry)
      - Navigate to the same slug used in test (c)
      - Verify the content is still there
      - NOTE: Since tests run sequentially in a single file, use `test.describe.serial` and share the slug across tests (c) and (d) via a variable declared at describe scope

   e. **"websocket connects for real-time mode"**
      - Create a pad via API POST, then PATCH to set mode to 'real-time'
      - Navigate to the pad page
      - Wait for the connection dot (`.connection-dot`) to be visible -- this proves WebSocket connected through Docker port mapping
      - Alternatively, open a WebSocket directly to `ws://localhost:$PORT/ws/pads/$slug` and verify the connection opens

   **Important:** The test file must NOT rely on `playwright.config.ts` webServer block. Use `test.use({ baseURL: process.env.DOCKER_TEST_URL || 'http://localhost:3198' })` at the describe level.
  </action>
