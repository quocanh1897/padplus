```typescript
async function select(value: typeof mode) {
  if (value === mode) { open = false; return; }
  try {
    const res = await fetch(`/api/pads/${slug}/mode`, {
      method: 'PATCH',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ mode: value })
    });
    if (res.ok) {
      onModeChange(value);
    }
  } catch {
    // Silently fail -- mode stays unchanged
  }
  open = false;
}
```

Wait -- re-reading CONTEXT.md: "When auto-merge fails (can't produce a clean result), fall back to the existing conflict banner from Phase 1 -- user chooses overwrite or copy-reload." And "Overlapping same-line edits: let the merge algorithm decide (best-effort)."

So the approach should be: when `hadConflicts` is true, the merge still produced a best-effort result that was saved to the server. The editor should show this merged content AND show the conflict banner so the user can review. The "overwrite" action on the conflict banner in this case would just dismiss the banner (content is already the merged result). The "copy and reload" would let the user copy their original content and reload.

Actually, simpler approach aligned with CONTEXT.md: if `hadConflicts` is true, show the conflict banner with the merged content as the "server" version and the user's original content as the "local" version. The user can choose to keep the merge result (dismiss) or copy their version and reload. This reuses the existing ConflictBanner without modification.
