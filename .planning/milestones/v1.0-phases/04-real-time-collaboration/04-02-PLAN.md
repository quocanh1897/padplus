2. Create `src/lib/components/RealtimeEditor.svelte`:
   - Props (Svelte 5 $props): `{ slug: string; initialContent: string; onStatusChange: (status: 'connected' | 'connecting' | 'disconnected') => void }`
   - State: `let textareaEl: HTMLTextAreaElement`, `let synced = $state(false)`
   - On mount (use `onMount`): dynamically import yjs, y-websocket, y-textarea:
     ```typescript
     const Y = await import('yjs');
     const { WebsocketProvider } = await import('y-websocket');
     const { TextAreaBinding } = await import('y-textarea');
     ```
   - Create Y.Doc, get Y.Text named 'content'
   - Create WebsocketProvider: `new WebsocketProvider(wsUrl, slug, doc, { connect: true, maxBackoffTime: 5000 })`
     - wsUrl: `${window.location.protocol === 'https:' ? 'wss:' : 'ws:'}//${window.location.host}/ws/pads`
   - Listen for provider 'status' event: map `event.status` to connection state. 'connected' -> 'connected', 'disconnected' -> check if wsconnecting then 'connecting' else 'disconnected'. Call onStatusChange.
   - Listen for provider 'sync' event: when `synced === true` AND binding not yet created, create `new TextAreaBinding(ytext, textareaEl)`. Set synced = true. This prevents content flicker (Research Pitfall 8).
   - Before binding: show initialContent in the textarea as read-only placeholder (SSR content preserved). Once synced, the binding takes over.
   - On destroy (use `onDestroy`): binding?.destroy(), provider.disconnect(), provider.destroy(), doc.destroy()
   - Template: single `<textarea bind:this={textareaEl} class="editor" value={initialContent} readonly={!synced} placeholder="Connecting...">`. The "editor" class ensures identical styling to the existing textarea (same CSS from +page.svelte styles). Spellcheck="true".
   - NO additional styling in this component -- it inherits the `.editor` class styles from the parent page. Only add a subtle opacity transition: when `!synced`, set `opacity: 0.7` to hint that sync is in progress.
